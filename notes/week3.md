# Week 3 — CI/CD Fundamentals

This week focused on understanding Continuous Integration and Continuous Delivery concepts.

Key concepts studied:

- Continuous Integration (CI)
- Continuous Delivery vs Continuous Deployment
- Why automated pipelines are important
- The role of CI systems in modern development workflows

Key ideas learned:

CI pipelines automate the process of building, testing, and validating software changes.

Instead of relying on manual steps, pipelines ensure that every commit goes through the same reproducible process.

Practical work:

- Designed the first draft of the CI pipeline
- Documented the pipeline structure in `pipeline-draft.md`
- Identified the stages needed for the project pipeline

Typical pipeline stages:

1. Build
2. Test
3. Validation
4. Artifact generation

Lessons learned:

CI pipelines improve software quality by validating every change automatically.
